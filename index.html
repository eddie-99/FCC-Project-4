Introducción al DOM

El modelo de objetos de documento ( DOM ) es la representación de datos de los objetos que componen la estructura y el contenido de un documento en la web. Esta guía presentará el DOM, verá cómo el DOM representa un documento HTML en la memoria y cómo utilizar las API para crear aplicaciones y contenido web.

¿Qué es el DOM?

El Modelo de objetos de documento (DOM) es una interfaz de programación para documentos web. Representa la página para que los programas puedan cambiar la estructura, el estilo y el contenido del documento. El DOM representa el documento como nodos y objetos; de esa manera, los lenguajes de programación pueden interactuar con la página.

Una página web es un documento que puede mostrarse en la ventana del navegador o como fuente HTML. En ambos casos, es el mismo documento pero la representación del Modelo de Objetos de Documento (DOM) permite manipularlo. Como representación orientada a objetos de la página web, se puede modificar con un lenguaje de programación como JavaScript.

Por ejemplo, DOM especifica que el querySelectorAllmétodo en este fragmento de código debe devolver una lista de todos los < p > elementos del documento:

const paragraphs = document.querySelectorAll("p"); // paragraphs[0] is the first < p > element // paragraphs[1] is the second < p > element, etc. alert(paragraphs[0].nodeName);

Todas las propiedades, métodos y eventos disponibles para manipular y crear páginas web están organizados en objetos. Por ejemplo, el documentobjeto que representa el documento en sí, cualquier tableobjeto que implemente la HTMLTableElementinterfaz DOM para acceder a tablas HTML, etc., son todos objetos.

El DOM se construye utilizando varias API que funcionan juntas. El DOM principal define las entidades que describen cualquier documento y los objetos que contiene. Esto se amplía según sea necesario por otras API que agregan nuevas características y capacidades al DOM. Por ejemplo, la API DOM HTML agrega soporte para representar documentos HTML en el DOM principal, y la API SVG agrega soporte para representar documentos SVG.

DOM y JavaScript

El breve ejemplo anterior, como casi todos los ejemplos, es JavaScript . Es decir, está escrito en JavaScript, pero usa el DOM para acceder al documento y sus elementos. El DOM no es un lenguaje de programación, pero sin él, el lenguaje JavaScript no tendría ningún modelo o noción de páginas web, documentos HTML, documentos SVG y sus componentes. El documento en su conjunto, el encabezado, las tablas dentro del documento, los encabezados de las tablas, el texto dentro de las celdas de la tabla y todos los demás elementos de un documento son partes del modelo de objetos de documento para ese documento. Se puede acceder a todos ellos y manipularlos utilizando el DOM y un lenguaje de programación como JavaScript.

El DOM no forma parte del lenguaje JavaScript, sino que es una API web que se utiliza para crear sitios web. JavaScript también se puede utilizar en otros contextos. Por ejemplo, Node.js ejecuta programas JavaScript en una computadora, pero proporciona un conjunto diferente de API, y la API DOM no es una parte central del tiempo de ejecución de Node.js.

El DOM fue diseñado para ser independiente de cualquier lenguaje de programación en particular, haciendo que la representación estructural del documento esté disponible desde una API única y consistente. Incluso si la mayoría de los desarrolladores web solo usarán el DOM a través de JavaScript, las implementaciones del DOM se pueden construir para cualquier lenguaje, como lo demuestra este ejemplo de Python:

# Python DOM example import xml.dom.minidom as m doc = m.parse(r"C:\Projects\Py\chap1.xml") doc.nodeName # DOM property of document object p_list = doc.getElementsByTagName("para")

Para obtener más información sobre las tecnologías involucradas en la escritura de JavaScript en la web, consulte Descripción general de las tecnologías de JavaScript .

Accediendo al DOM

No tiene que hacer nada especial para comenzar a usar el DOM. Utiliza la API directamente en JavaScript desde dentro de lo que se llama un script , un programa ejecutado por un navegador.

Cuando crea un script, ya sea en línea en un script elemento o incluido en la página web, puede comenzar inmediatamente a usar la API para los objetos documentu windowpara manipular el documento en sí, o cualquiera de los diversos elementos en la página web (los elementos descendientes de el documento). Su programación DOM puede ser algo tan simple como el siguiente ejemplo, que muestra un mensaje en la consola usando la console.log()función:

< body > onload="console.log('Welcome to my home page!');"> < /body >

Como generalmente no se recomienda mezclar la estructura de la página (escrita en HTML) y la manipulación del DOM (escrito en JavaScript), las partes de JavaScript se agruparán aquí y se separarán del HTML.

Por ejemplo, la siguiente función crea un nuevo < h1 >elemento, agrega texto a ese elemento y luego lo agrega al árbol del documento:

// run this function when the document is loaded window.onload = function() { // create a couple of elements in an otherwise empty HTML page const heading = document.createElement("h1"); const heading_text = document.createTextNode("Big Head!"); heading.appendChild(heading_text); document.body.appendChild(heading); }
Tipos de datos fundamentales

Esta página intenta describir los diversos objetos y tipos en términos simples. Pero hay varios tipos de datos diferentes que se transmiten a través de la API que debe conocer.

Nota: Debido a que la gran mayoría del código que usa DOM gira en torno a la manipulación de documentos HTML, es común referirse a los nodos en DOM como elementos , aunque estrictamente hablando no todos los nodos son elementos.

La siguiente tabla describe brevemente estos tipos de datos.

  1. Document
    • Cuando un miembro devuelve un objeto de tipo document(por ejemplo, la ownerDocumentpropiedad de un elemento devuelve el objeto documental que pertenece), este objeto es el documentobjeto raíz en sí. El capítulo de referencia DOMdocument describe el documentobjeto.
  2. Node
    • Cada objeto ubicado dentro de un documento es un nodo de algún tipo. En un documento HTML, un objeto puede ser un nodo de elemento pero también un nodo de texto o un nodo de atributo.
  3. Element
    • El elementtipo se basa en node. Se refiere a un elemento o nodo de tipo elementdevuelto por un miembro de la API DOM. En lugar de decir, por ejemplo, que el document.createElement()método devuelve una referencia de objeto a a node, simplemente decimos que este método devuelve el elementque se acaba de crear en el DOM. elementLos objetos implementan la Elementinterfaz DOM y también la Nodeinterfaz más básica , las cuales se incluyen juntas en esta referencia. En un documento HTML, los elementos se mejoran aún más mediante la HTMLElementinterfaz HTML DOM API, así como otras interfaces que describen las capacidades de tipos específicos de elementos.
  4. NodeList
    • A nodeListes una matriz de elementos, como el tipo que devuelve el método document.querySelectorAll(). Elementos de una nodeListse accede por el índice de una de dos maneras:
      - list.item (1)
      - lista [1]
      Estos dos son equivalentes. En el primero, item()está el método único sobre el nodeListobjeto. Este último usa la sintaxis de matriz típica para buscar el segundo elemento de la lista.
  5. Attr
    • Cuando un attributemiembro (por ejemplo, el createAttribute()método) devuelve an , es una referencia de objeto que expone una interfaz especial (aunque pequeña) para los atributos. Los atributos son nodos en el DOM al igual que los elementos, aunque es posible que rara vez los use como tales.
  6. NamedNodeMap
    • A namedNodeMapes como una matriz, pero se accede a los elementos por nombre o índice, aunque este último caso es simplemente una conveniencia para la enumeración, ya que no están en un orden particular en la lista. A namedNodeMaptiene un item()método para este propósito, y también puede agregar y eliminar elementos de un namedNodeMap.
